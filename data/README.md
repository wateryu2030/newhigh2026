# 策略数据目录说明

本目录存放各策略依赖的 CSV 数据文件。部分文件可由 Web 回测时**自动补齐**（见 `database/auto_fix_strategy_data.py`），也可手动编辑或通过脚本更新。

## 文件用途与更新方式

| 文件 | 使用策略 | 说明 | 更新建议 |
|------|----------|------|----------|
| **industry_stock_map.csv** | 行业轮动 (strategy1) | 行业–股票映射：每行一只股票，列 `代码`、`行业名称`。用户在选择股票回测时，该股票会被自动加入此表。 | 可手动增删行；或从自选/板块导出后合并。 |
| **industry_score.csv** | 行业轮动 (strategy1) | 行业得分表：索引为行业名称，列含 `综合得分`（及可选 `景气度得分`、`资金流得分`）。用于月度选行业。 | 建议每月或每季根据景气度/资金流数据更新得分。 |
| **tech_leader_stocks.csv** | 动量+均值回归 (strategy2)、财报超预期 (strategy3) | 高科技龙头股票池：列 `代码`、`名称`。用户选股回测时会自动加入。 | 可手动维护龙头列表，或从行业成分股导出。 |
| **consume_leader_stocks.csv** | 动量+均值回归 (strategy2)、财报超预期 (strategy3) | 消费龙头股票池：列 `代码`、`名称`。 | 同上。 |
| **earnings_events.csv** | 财报超预期 (strategy3) | 财报事件表：列如 `股票代码`、`事件日期`、`事件类型`。当前为示例结构，实际需对接财报/预期数据。 | 需从财报或第三方预期数据定期更新。 |
| **etf_list.csv** | ETF 网格 (strategy4) | ETF 列表：列 `代码`、`名称`。策略内另有硬编码的 ETF 代码，可与此表配合。 | 按需更新 ETF 代码与名称。 |

## 数据库

- **astock.db**：SQLite 数据库，存放日线行情等，由「同步选中股票数据」或数据拉取脚本写入。回测时优先使用库内数据。

## 注意

- 编码请使用 **UTF-8（带 BOM 时用 utf-8-sig）**，以保证中文与 Web 展示正常。
- 修改 CSV 后无需重启服务，下次回测会读取最新文件。
- 本平台仅用于学习与回测，不构成投资建议。
