## 三、对反量化策略的影响

反量化长线选股策略依赖 5 年十大股东数据计算：

| 因子 | 依赖条件 | 当前可算性 |
|------|----------|------------|
| `top10_ratio` | 单期即可 | ✅ 可算 |
| `top10_ratio_std` | 至少 2 个报告期 | ❌ 全为 NaN |
| `long_term_institution_count` | 连续 8 季度出现 | ❌ 全为 0 |
| `institution_count_current` | 单期机构数 | ✅ 可算 |
| `turnover_avg` | 相邻报告期对比 | ❌ 全为 0 |

**结论**：当前数据仅能支撑「放宽模式」（持股集中度 + 当前机构数），无法实现完整 5 年反量化策略。

## 四、采集逻辑原因

`data/src/data/collectors/financial_report.py` 中：

```python
# save_top_10_shareholders 关键逻辑：
report_date_val = df[date_col].iloc[0]   # 仅取第一行日期
df_top10 = df.head(10)                    # 仅取前 10 行
# INSERT OR REPLACE ... PRIMARY KEY (stock_code, report_date, rank)
```

- `ak.stock_main_stock_holder` 按文档可返回**多期历史**，但采集器只用了 `iloc[0]` 的日期
- `INSERT OR REPLACE` 按 (stock_code, report_date, rank) 主键写入，同股票不同 report_date 本可并存
- 实际只写入了一期：要么 API 只返回最新一期，要么多期被混在一起、按单期处理
- 需验证：调用接口查看返回的 `截至日期` 是否有多个不同值；若有，应按日期分组逐期写入

## 五、补齐建议

### 1. 评估 AkShare 历史股东接口

- 查阅 `ak.stock_main_stock_holder` 是否支持多期（如通过 report_date 参数）
- 若有，需改动采集逻辑，按报告期循环拉取并写入多期数据

### 2. 使用 Tushare（需 token）

Tushare 提供 `top10_floatholders` 等历史十大股东接口，可拉取多年度、多季度数据，适合做反量化策略。

### 3. 采集改造要点

- 每只股票按 2021Q1–2025Q4 等报告期循环采集
- 单股多期数据写入同一 `top_10_shareholders` 表，不覆盖历史
- 全量补采预估：约 5,000 股 × 20 期 ≈ 10 万次请求，需控制频率和 token 限制

## 六、短期可做

在当前只有单期数据的前提下：

1. **继续使用放宽模式**：持股集中度 ≥50%、机构数 ≥2，可筛出约 800 只候选
2. **等待/接入历史数据**：完成采集改造后再启用完整严格模式
