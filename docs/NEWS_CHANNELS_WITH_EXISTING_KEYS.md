# 抖音 / X 等渠道 × 现有 API Key：怎么接

## 先讲清楚边界

| 渠道 | 能否用「一个通用 Key」直接拉原始数据？ | 现实做法 |
|------|----------------------------------------|----------|
| **抖音** | **不能**。抖音公开数据需字节开放平台资质，无「一个 Key 读全站」的开放接口。 | 用 **豆包（火山 Ark）**：同一公司生态，在方舟里给接入点开 **联网内容插件** 后，模型回答可融合其能检索到的公开网络信息（是否覆盖抖音站内取决于平台策略，非爬虫保证）。 |
| **X (Twitter)** | **不能**用你现在的 Key。 | 需另配 **X API**（收费档位）或 **xAI Grok API**，或 **Gemini + Google Search Grounding**（`GEMINI_API_KEY`）。 |
| **微信公众号** | 无面向第三方的全文检索 API。 | **通义 / 豆包联网** 检索公开网页上的转载与报道；或腾讯侧产品（元宝）需单独账号，不在当前 `.env` 里。 |

结论：**现有 `.env` 里能做的**，是走 **大模型 + 官方「联网」能力**，让模型代你「搜一圈再答」，而不是自己拿 Key 去调抖音/X 的私有数据 API。

---

## 与本项目已对齐的配置（无需新 Key）

| 变量 | 用途 |
|------|------|
| `DOUBAO_API_KEY` + `DOUBAO_MODEL`（推理接入点 ID） | 火山方舟 Chat；**控制台开启联网插件**后，适合中文热点、字节系内容理解类问题。 |
| `DASHSCOPE_API_KEY` / `BAILIAN_API_KEY` | 通义兼容接口；若百炼应用侧开启联网/搜索类能力，同理可增强时效性。 |
| `GEMINI_API_KEY` | 若启用 **Google Search Grounding**，偏全球网页与英文社交公开讨论。 |

---

## 你要做的配置步骤（豆包 → 接近「抖音/中文热点」）

1. 登录 [火山引擎方舟](https://console.volcengine.com/ark) → 你的 **推理接入点**（与 `DOUBAO_MODEL=ep-...` 对应）。  
2. 按官方文档为该接入点配置 **联网内容插件**（原「联网搜索」已升级，见 [联网搜索 / 联网内容插件](https://www.volcengine.com/docs/82379/1756990)）。  
3. 保存后，调用本仓库 **`POST /api/news/web-insight`**，`provider: "doubao"`，把问题写成例如：「最近一周某某概念在抖音/中文互联网上的讨论热点摘要」。  

未开联网时，接口仍会返回模型回答，但会偏训练截止知识，需在文案里说明「未启用联网」。

---

## X / 海外舆情（需额外 Key）

1. **Grok / xAI**：注册 xAI API，在 `.env` 增加例如 `XAI_API_KEY`，后续可在 Gateway 加 `provider: "grok"`（待实现）。  
2. **X API**：开发者门户申请，按条计费。  
3. **Gemini Grounding**：在 Google AI Studio 启用带搜索的模型能力后，可用 `GEMINI_API_KEY` 做英文/全球网页向摘要。

---

## 与本项目 API 的关系

- **`GET /api/news`**：仍是东财 + DuckDB，**不接抖音**。  
- **`POST /api/news/web-insight`**（新增）：用现有豆包/通义 Key，**自然语言问「热点/舆情」**；是否「真联网」取决于你在火山/百炼控制台的插件开关。  

更重的 **浏览器自动化多站**（AI-Search-Hub）仍建议独立跑，见 `docs/NEWS_SEARCH_AI_SEARCH_HUB.md`。
